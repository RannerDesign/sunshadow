Shadow Simulation

The application sunshadow is designed to calculate shadow of given objects based on the sun position at date and time to be entered.

Configurations can be saved as files in json format. Date and time can be entered by keying in or using range dials. Once the simulation is started, sun position at that day and time is shown as azimuth (0° = south) and altitude (0 - 90°).

The content of the configuration file can be edited in the bottom part of the window. Be careful to comply with the json syntax. Errors will be marked by the syntax highlighting editor (ACE).

Structure of the configuration file (json format):

Arbitrary text to identify configuration
Date and time of stored configuration
Geoposition longitude of the configuration
Geoposition latitude of the configuration
Orientation of the y-axis of the coordinated (0° = north)
Min level for sun altitude, below is regarded as darkness
Length units of all coordinates
z-coordinate level for which shadows are calculated (e.g. 0 = ground floor, 720 mm may be a table)
if true, force same scaling factors for X and Y axis by changing canvas height
Array defining the area for which shadows are calculated and shown
[xmin, xmax, ymin, ymax]
Definition of the shadow generating objects
List with elements containing the following fields:
Activate or deactivate element (true, false) for shadow generation
Activate or deactivate element (true, false) for drawing
Object type, describing the way coordinates are given, the following values are allowed:
  • cuboid: defined by param
  • poly: defined by polynom data
Kind of material, allowed values are:
  • wall
  • canopy
  • fabric
  • vertical
  • draw
Descriptive text for this element
xmin, xmax, ymin, ymax, bottom, top
polygon defined by a list of [x, y, z] coordinates (polygon closed automatically from last to fist list element)
json object containing canvas formatting elements (e.g. strokeStyle, fillStyle)
